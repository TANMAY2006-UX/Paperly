## Task 4 — Category Explanations

### `INLINE_BOLD`
*   **Why Landmark emitted it**: Inline bold phrases have a higher `typography_class` than the non-bold text preceding and following them, creating a mathematical local maximum.
*   **Compiler stage owner**: Assembly (for providing bounds) / Semantic (for failing to verify baseline geometry).
*   **Behavior**: **Incorrect**. Semantic fails to reject these because it ignores the bounding boxes that prove they are inline.

### `AUTHOR`
*   **Why Landmark emitted it**: Author names are often printed in a slightly larger or bolder font than the subsequent affiliations, creating a localized peak.
*   **Compiler stage owner**: Semantic (Front Matter).
*   **Behavior**: **Architectural limitation**. Landmark correctly executes its math, but the architecture inappropriately forces Semantic to rely on `is_outlier` to exit front matter, causing authors to trigger state transitions erroneously.

### `TABLE_VALUE`
*   **Why Landmark emitted it**: Numerical values inside tables are sometimes bolded to indicate best results, creating isolated typographic peaks.
*   **Compiler stage owner**: Semantic.
*   **Behavior**: **Incorrect**. The parser currently lacks a structural bounding box filter to realize the text is part of a tabular grid.
