# ADR-002 — Responsibility Ownership Audit
*Architectural Validation of the Semantic Parser*

## Part 1 — Responsibilities Inside `semantic.py`

Based on the frozen architecture and code traces, `semantic.py` currently performs the following operations:
1.  **Universal Body Class Computation**: Calculating the dominant typography class between the Title and References.
2.  **Structural Candidate Selection**: Deciding which groups are allowed to be evaluated as non-paragraph objects (currently delegated to `is_outlier`).
3.  **Parser State Transition**: Moving the state machine (e.g., from `BODY_PARSE` to `REFERENCES_PARSE`).
4.  **Lexical Verification**: Applying regular expressions to validate specific text (e.g., "Abstract", "References", "1. Introduction").
5.  **Section Stack Management**: Comparing typography magnitude to push/pop hierarchical scope.
6.  **Semantic Node Creation**: Instantiating specific objects like `SECTION_HEADER`, `AUTHORS`, `CAPTION`.
7.  **Hierarchy Construction**: Assembling the final semantic DAG.

---

## Part 2 — Required Information per Responsibility

*   **Universal Body Class**: Requires `Typography` (class ID), `Lexical Text` (to ignore captions), and `Anchor Endpoints` (Title/Ref IDs).
*   **Structural Candidate Selection**: Requires `Typography` (to ignore UBC) and `Parser State`.
*   **Parser State Transition**: Requires `Parser State` and `Lexical Text` (Regex for "References").
*   **Lexical Verification**: Requires `Lexical Text`.
*   **Section Stack Management**: Requires `Hierarchy` (current stack depth) and `Typography` (magnitude comparison).
*   **Semantic Node Creation**: Requires `Parser State` and `Lexical Verification`.

---

## Part 3 — Architectural Ownership

Based strictly on the architecture documentation:
*   **Assembly**: Owns physical typography clustering and vector generation.
*   **Landmarks**: Owns global page anchoring (finding the absolute start (Title) and absolute end (References) of the document content).
*   **Semantic**: Owns parsing context, state transitions, structural validation, and hierarchy resolution.

---

## Part 4 — Responsibility Violations

**Violation 1: Structural Candidate Selection**
*   **Current Owner:** Landmark (via `is_outlier` mathematical local maxima).
*   **Correct Owner:** Semantic (via `typography_class != universal_body_class`).
*   **Evidence:** PIP-003, EXP-06, EXP-10.
*   **Architectural Consequence:** Landmark attempts to do Semantic's job using only physics. Because Landmark lacks text and state, it fails catastrophically, discarding 70% of structural objects before Semantic can even look at them.

---

## Part 5 — Multiplexed Responsibilities

**Multiplexed Variable:** `is_outlier`
Inside `semantic.py` (`BODY_PARSE`), the `if is_outlier:` check controls **two entirely different architectural responsibilities** simultaneously:

1.  **Parser State Transition:** (Lines 140-145) If `is_outlier` is true, check if the text says "References" to change the parser state.
2.  **Structural Candidate Selection:** (Line 150) If the text didn't trigger a state transition, the `else` block blindly assumes the `is_outlier` flag implies a Valid Section Header and forces a node creation.

---

## Part 6 — Separation Feasibility

Can `is_outlier` be de-multiplexed **without changing architecture?**
**Yes.** 
Separating state transitions from candidate selection is a purely localized refactoring of the `if/else` control flow inside `semantic.py`. It requires no redesign of the Layout, Assembly, or Landmark stages, and it explicitly aligns the code with the intended architectural contract (Semantic owns state, Landmark provides anchors).

---

---

## Part 8 — Final Verdict

**1. Is the proposed localized refactoring architecturally valid?**
Yes. Reassigning Candidate Selection from Landmark (physics) to Semantic (context + text) restores the original architectural contract. Landmark should only anchor the document; Semantic must parse it.

**2. Is there exactly one responsibility that should be refactored first?**
Yes.

**3. If yes, which one?**
**De-multiplexing State Transitions from Candidate Selection inside `semantic.py`.** 

We cannot safely implement the EXP-10 structural fix (modifying candidate selection) until we decouple the State Transition logic from the `if is_outlier:` block. Refactoring the `BODY_PARSE` control flow to separate these two responsibilities is the mandatory first step.
